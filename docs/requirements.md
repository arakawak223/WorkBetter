# WorkBetter - 求人アグリゲーションアプリ 要件定義書

## 1. プロジェクト概要

**アプリ名:** WorkBetter

**概要:** 転職希望者が条件（年齢・職種・勤務地・年収）を入力するだけで、ウェブ上の求人情報を集約し、最適な求人リストを瞬時に生成するプラットフォーム。

**ターゲットユーザー:** 日本国内の転職希望者

---

## 2. 機能要件

### 2.1 検索条件入力（フロントエンド）

| 項目 | 入力形式 | 詳細 |
|------|---------|------|
| 年齢 | 数値入力 | 求人の対象年齢マッチングに使用 |
| 希望職種 | フリーワード + カテゴリ選択 | カテゴリは事前定義、フリーワードで柔軟に対応 |
| 希望勤務地 | 都道府県 / 市区町村 / リモート可否 | ドロップダウン + チェックボックス |
| 希望年収 | 下限・上限スライダー | 万円単位、50万円刻み |

### 2.2 データ収集エンジン（バックエンド）

- **求人API連携:** 公開APIを持つ求人サービスからデータ取得
  - 対象: 求人ボックス、スタンバイ、Indeed API 等（利用規約に準拠するもの）
- **スクレイピング:** APIが無い場合、robots.txt を遵守した上でのクローリング
- **収集頻度:** 日次バッチ + ユーザーリクエスト時のリアルタイム検索

### 2.3 データ加工・クレンジング

- **重複排除:** 企業名 + 職種名 + 勤務地のハッシュで同一案件を統合
- **構造化:** LLM（自然言語処理）を用いて非構造データから以下を抽出
  - 給与範囲（固定給 / インセンティブ / 賞与）
  - 募集期限
  - 必須スキル・経験年数
- **正規化:** 勤務地名・職種名の表記ゆれ統一

### 2.4 抽出項目（アウトプット）

各求人に対して以下の情報を表示:

| 項目 | 説明 |
|------|------|
| 企業名 | 募集元の企業名 |
| 職種名 | ポジション名 |
| 勤務地詳細 | 所在地 + リモート可否 |
| 仕事内容 | AI生成のサマリー（100文字程度） |
| 給与条件 | 年収レンジ、手当、賞与 |
| 募集期限 | 掲載終了日 |
| 詳細URL | 元サイトへのリンク |
| 情報元区分 | 企業直接掲載 / エージェント経由 を示すフラグ |
| マッチ度 | ユーザー条件との適合スコア（%） |

### 2.5 リスト表示・管理

- **一覧表示:** カード形式で求人を表示
- **ソート:** 年収順 / マッチ度順 / 募集期限順 / 新着順
- **フィルタ:** 追加条件での絞り込み（雇用形態、企業規模、情報元区分等）
- **お気に入り:** 気になる求人のブックマーク機能
- **履歴:** 過去の検索条件の保存・再利用

### 2.6 ユーザー管理

- **認証:** メールアドレス + パスワード / ソーシャルログイン（Google）
- **プロフィール:** 基本情報の保存（検索条件のデフォルト値として利用）

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| 法的遵守 | robots.txt 確認、利用規約遵守、レートリミット遵守 |
| パフォーマンス | 検索結果表示まで3秒以内（キャッシュ利用時） |
| スケーラビリティ | 同時100ユーザーを初期目標、クラウドインフラで水平スケール可能 |
| セキュリティ | ユーザー情報の暗号化、HTTPS必須、OWASP Top 10 対策 |
| データ鮮度 | 日次クロールで募集終了案件を自動非表示 |
| 可用性 | 99.5% SLA（初期フェーズ） |

## 4. 画面構成

### 4.1 画面一覧

1. **ランディングページ** - サービス説明 + 検索フォーム
2. **検索入力画面** - 4条件の入力フォーム
3. **解析中画面** - ローディング + 進捗表示
4. **検索結果一覧** - カード形式の求人リスト
5. **求人詳細画面** - 選択した求人の全情報表示
6. **お気に入り一覧** - ブックマークした求人
7. **ログイン / 新規登録** - 認証画面
8. **マイページ** - プロフィール、検索履歴

### 4.2 画面遷移

```
ランディングページ
  |-> ログイン / 新規登録 -> マイページ
  |-> 検索入力画面
        |-> 解析中画面
              |-> 検索結果一覧
                    |-> 求人詳細画面
                    |-> お気に入りに追加
  |-> お気に入り一覧
```

## 5. データモデル（主要エンティティ）

### users
- id, email, password_hash, name, age, created_at, updated_at

### search_conditions
- id, user_id, age, occupation, location, salary_min, salary_max, remote_ok, created_at

### jobs
- id, company_name, title, location, description, salary_min, salary_max, employment_type, source_type (ENUM: 'direct' | 'agent'), agent_name (nullable), source_url, source_site, expires_at, created_at, updated_at

**source_type の判定ルール:**
- `direct`: 企業の採用ページや企業が直接掲載した求人
- `agent`: 人材紹介会社・転職エージェント経由の求人（リクルートエージェント、doda、JACリクルートメント等）
- 判定方法: 収集元サイトのドメイン・掲載パターンで自動分類 + LLMによる求人文面からの推定

### favorites
- id, user_id, job_id, created_at

### job_hashes
- id, job_id, dedup_hash (重複排除用)
